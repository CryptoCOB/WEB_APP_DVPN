Web Application Overview

Project Type: Custom Secure Communication Platform (DVPN)

Summary:

DVPN Dynamics is a web-based simulation of a decentralized virtual private network. Unlike traditional VPNs that rely on a central server, our application demonstrates how data can be securely routed through multiple nodes (Toronto → Halifax) to ensure anonymity and data integrity.

Core Value: Privacy, Security, and Decentralization.

Project Planning

Planned Features

  • ✅ User Authentication: Secure Login/Register with JWT.
  • ✅ Multi-hop Routing: Simulation of data passing through nodes.
  • ✅ Encryption: AES-256-CBC for payload protection.
  • ✅ Logging: MongoDB storage for transaction history.

Planning Challenges

Features Not Implemented

  • ❌ True P2P Networking:
    Reason: Complexity of NAT traversal and time constraints. We simulated nodes as API endpoints instead.
  • ❌ Dynamic Node Discovery:
    Reason: Requires a Distributed Hash Table (DHT). We used static routing for the prototype.

Execution Challenges

  • Handling asynchronous encryption/decryption flows.
  • Ensuring consistent state across simulated nodes.

Instructions

How to use DVPN Dynamics

1. Access & Authentication

  1. Navigate to the Login Page.
  2. Click "Register" to create a new account.
  3. Enter your credentials to access the Dashboard.

POST /api/auth/login
{
  "username": "user1",
  "password": "password123"
}

2. Sending Secure Data

  1. On the Dashboard, locate the "New Packet" form.
  2. Enter the destination IP and your message payload.
  3. Click "Route Packet".
  4. The system encrypts the data and forwards it to the Toronto Node.

3. Viewing Logs

  1. Navigate to the "Transmission Logs" section.
  2. View the history of sent packets.
  3. Verify the status (Encrypted/Decrypted) and timestamps.

If We Started Again...
